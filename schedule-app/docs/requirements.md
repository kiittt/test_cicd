# 日程安排 CRUD 需求文档

## 1. 项目背景

本项目用于验证一套完整的软件测试与 CI/CD 流程。业务选择“日程安排 CRUD”，范围小但链路完整，适合覆盖前端、后端、数据库、自动化测试和质量门禁。

## 2. 用户角色

| 角色 | 说明 |
| --- | --- |
| 普通用户 | 管理个人日程，支持新增、查询、编辑和删除 |

## 3. 功能范围

### 3.1 新增日程

用户可以创建一条日程，填写标题、描述、开始时间、结束时间、地点、优先级和状态。

### 3.2 查询日程

用户可以查看日程列表，并支持按关键字、日期和状态筛选。

### 3.3 查看详情

用户可以查看单条日程的完整信息。

### 3.4 编辑日程

用户可以修改已存在日程的标题、描述、时间、地点、优先级和状态。

### 3.5 删除日程

用户可以删除指定日程，删除后该日程不再出现在列表中。

## 4. 字段定义

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | number | 是 | 日程唯一标识，由系统生成 |
| title | string | 是 | 日程标题，不能为空 |
| description | string | 否 | 日程描述 |
| startTime | datetime | 是 | 开始时间 |
| endTime | datetime | 是 | 结束时间 |
| location | string | 否 | 地点 |
| priority | string | 是 | 优先级：low、medium、high |
| status | string | 是 | 状态：todo、done、cancelled |
| createdAt | datetime | 是 | 创建时间，由系统生成 |
| updatedAt | datetime | 是 | 更新时间，由系统生成 |

## 5. 业务规则

| 编号 | 规则 |
| --- | --- |
| R001 | 标题不能为空 |
| R002 | 开始时间不能晚于结束时间 |
| R003 | 优先级只能是 `low`、`medium`、`high` |
| R004 | 状态只能是 `todo`、`done`、`cancelled` |
| R005 | 查询不存在的日程时返回 404 |
| R006 | 删除不存在的日程时返回 404 |
| R007 | 编辑不存在的日程时返回 404 |

## 6. API 设计

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/health` | 健康检查 |
| GET | `/api/schedules` | 查询日程列表 |
| GET | `/api/schedules/:id` | 查询日程详情 |
| POST | `/api/schedules` | 新增日程 |
| PUT | `/api/schedules/:id` | 编辑日程 |
| DELETE | `/api/schedules/:id` | 删除日程 |

## 7. 查询参数

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| keyword | 按标题或描述模糊查询 | `keyword=会议` |
| status | 按状态筛选 | `status=todo` |
| date | 按日期筛选 | `date=2026-07-01` |

## 8. 错误响应

```json
{
  "message": "title is required"
}
```

| 场景 | 状态码 |
| --- | --- |
| 参数错误 | 400 |
| 资源不存在 | 404 |
| 方法不允许 | 405 |
| 服务异常 | 500 |

## 9. 核心异常场景

- 标题为空。
- 标题超长。
- 开始时间晚于结束时间。
- 优先级非法。
- 状态非法。
- 查询不存在的日程。
- 编辑不存在的日程。
- 删除不存在的日程。
- 查询结果为空。
- 网络异常或服务端异常。

## 10. 验收标准

- 用户可以通过页面完成新增、查询、编辑、删除。
- 后端接口符合 API 设计。
- 核心业务规则有自动化测试覆盖。
- CI 中自动执行 lint、单元测试、API 测试、UI 测试。
- CI 失败时不能合并到主分支。
